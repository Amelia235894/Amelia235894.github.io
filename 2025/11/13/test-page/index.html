学习之前

在今天学习之前,昨晚想了半天我应该从哪开始,学哪个可能会在兴奋点上,毕竟要尝试一下新看见的”方法论”,也是想了半天,学html吧,理由是

  1. html内容比较简单,不会有太多学不会的心理负担,且效果明显。
  2. 大一刚入学在ifLab学过一些CSS的内容,算是有点基础。
  3. 数据爬取的内容和这个也有关系(老师上次也提到过),诚然当时的代码大部分都是deepseek写的,顶多做点修改。当时找照片是在开发者工具里通过选取元素的方式找到类似于(img src=””)这种的,一般src里就是图片的url,不过我直接把这个元素复制给AI了,查找的时候似乎是查找的img的src属性,有点不记得了,反正当时基本修改输入输出路径就行,没有太大问题,除了ip被封那个。
  4. 之前一直也想过做一个个人网站,但是一直用git、node.js和hexo整完之后简单写一下就又重装系统了(windows重装过好几次),后面也就没再搞了。

我之前的学习呢,虽然也听进去了兴趣驱动,但是,但是,我看三维重建会把计算摄影学、计算图形学、论文巴拉巴拉一堆东西都下载下来然后说看,结果光下载就已经耗费了很多精力,一看又有点枯燥,就不想学了。看文献也是,英语是中等水平但是有点犟,就想看英文,但是又很耗时间,也容易困,可能后边能调整一下吧。

顺便提个好玩的,用豆包创建了一个黄老师的智能体,有点差劲,远不及黄老师的感觉哈哈哈哈,但是比较细致、及时,时不时还有(抬手看了眼腕表,语气放缓几分)(目光温和的看向你)这种类似的,有点意思。呜呜可能平时给黄老师消息发太多了……

开始学了

先问了HAI(那个智能体)一些琐碎的点,大概清了下思路。然后上午在MDN上学了配置环境等,虽然简单,但是也了解或补充了点之前不太重视或者已经知道的东西,如prettier一个前端代码规范化的工具,网站的规范结构组成有哪些:

  • scripts(文件夹),存放用户交互文件,如main.js
  • styles(文件夹),存放样式文件,如style.css
  • index.html,访问首页面
  • images(文件夹),存放网页所用图片

也对html,css,javascript了解多了一点,分别对应上面的前三项的内容,文件命名规则也再看了一遍(不用空格,尽量用小写字母和连字符,不用下划线等等),以及一些终端命令行,curl之前一直用,但是没了解过,今天简单了解了一下,顺便无意发现了一个使用软件/工具的文档阅读方法,有一些收获。

收获主要有以下四点(老师好像也都提过):

  1. 阅读官方文档,现在网上有很多别人写的csdn或者其他博客文档,但是都是二手的,不如官方文档更正确和权威,官方文档是第一手资料。
  2. 阅读官方文档有不懂的不要紧,先整体大概流程看一遍,有个大概的框架印象,然后再多看几遍。
  3. 多实践,光看很难起到作用,要跟着思考和实践,比如说我学prettier,要下载下来实际体验一下,试一试效果,实践出真知。
  4. 读懂文档且实践之后过一段时间可以再回过头来看文档,这个时候会有更高一级的收获,当时不明白的可能现在明白了,可能会有顿悟的感觉。

下午主要根据这几个html创建内容,CSS基础,JavaScript基础文档创建了一个最最最简单的网站(不过只有一个网页),网页的内容介绍如下:

html部分，html全称是hyper text makrup lanugage，超文本标记语言，由很多html元素组成，比如标题，段落，列表和链接，还有图片，这一块都是基本静态展示的部分，除了链接会跳转一下。html的内容就是当前页面能看见的所有文本和图片，这是一个网页最基本的内容，不过没有css的话没有样式，就类似于记事本的感觉。

style部分，顾名思义是样式，也就是html展示内容的样式，比较好理解的有字号多大，字体，颜色，位置是居中还是其他。可能不那么好理解的是每个元素都是一个盒子，有外边距margin，填充border和内边距padding，还有高度和宽度等等。反正加了这一部分之后可读性和效果会好很多。代码的规则也比较简单，要修改谁的样式，就把谁写前面，然后后边的属性用大括号括起来，不同属性分号分隔即可，不知道的属性到时候一搜索就行

scripts部分，涉及到交互，稍微复杂一点，javascript的变量声明不太一样，但是语法逻辑没有什么难度，感觉和之前搞Qt有点像。这个网页点击头像会在两张照片里进行切换，然后点击切换用户名字欢迎词也会跟着变。
